1405/*-------------------------------------------------------------------
1406// DIOI2C9AXISTRACKINGMPU9150::GetSlaveAddress
1407// Get the I2C address of the specified slave (0-3).
1408// Note that Bit 7 (MSB) controls read/write mode. If Bit 7 is set, it's a read
1409// operation, and if it is cleared, then it's a write operation. The remaining
1410// bits (6-0) are the 7-bit device address of the slave device.
1411//
1412// In read mode, the result of the read is placed in the lowest available
1413// EXT_SENS_DATA register. For further information regarding the allocation of
1414// read results, please refer to the EXT_SENS_DATA register description
1415// (Registers 73 - 96).
1416//
1417// The MPU-6050 supports a total of five slaves, but Slave 4 has unique
1418// characteristics, and so it has its own functions (getSlave4* and setSlave4*).
1419//
1420// I2C data transactions are performed at the Sample Rate, as defined in
1421// Register 25. The user is responsible for ensuring that I2C data transactions
1422// to and from each enabled Slave can be completed within a single period of the
1423// Sample Rate.
1424//
1425// The I2C slave access rate can be reduced relative to the Sample Rate. This
1426// reduced access rate is determined by I2C_MST_DLY (Register 52). Whether a
1427// slave's access rate is reduced relative to the Sample Rate is determined by
1428// I2C_MST_DELAY_CTRL (Register 103).
1429//
1430// The processing order for the slaves is fixed. The sequence followed for
1431// processing the slaves is Slave 0, Slave 1, Slave 2, Slave 3 and Slave 4. If a
