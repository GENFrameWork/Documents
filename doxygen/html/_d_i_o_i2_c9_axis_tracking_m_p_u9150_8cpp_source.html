1398/*-------------------------------------------------------------------
1399// DIOI2C9AXISTRACKINGMPU9150::GetSlaveAddress
1400// Get the I2C address of the specified slave (0-3).
1401// Note that Bit 7 (MSB) controls read/write mode. If Bit 7 is set, it's a read
1402// operation, and if it is cleared, then it's a write operation. The remaining
1403// bits (6-0) are the 7-bit device address of the slave device.
1404//
1405// In read mode, the result of the read is placed in the lowest available
1406// EXT_SENS_DATA register. For further information regarding the allocation of
1407// read results, please refer to the EXT_SENS_DATA register description
1408// (Registers 73 - 96).
1409//
1410// The MPU-6050 supports a total of five slaves, but Slave 4 has unique
1411// characteristics, and so it has its own functions (getSlave4* and setSlave4*).
1412//
1413// I2C data transactions are performed at the Sample Rate, as defined in
1414// Register 25. The user is responsible for ensuring that I2C data transactions
1415// to and from each enabled Slave can be completed within a single period of the
1416// Sample Rate.
1417//
1418// The I2C slave access rate can be reduced relative to the Sample Rate. This
1419// reduced access rate is determined by I2C_MST_DLY (Register 52). Whether a
1420// slave's access rate is reduced relative to the Sample Rate is determined by
1421// I2C_MST_DELAY_CTRL (Register 103).
1422//
1423// The processing order for the slaves is fixed. The sequence followed for
1424// processing the slaves is Slave 0, Slave 1, Slave 2, Slave 3 and Slave 4. If a
1425// particular Slave is disabled it will be skipped.
